# Customer Churn Dashboard

## Project Overview

This project analyzes customer churn behavior in a telecom company and prepares data for an interactive dashboard using Python.

The goal is to identify the key factors influencing customer churn and transform business insights into a dashboard that can support decision-making.

---

## Dataset Information

* Dataset: Telco Customer Churn
* Records: 7,043
* Features: 21

### Key Columns

* Churn
* Contract
* MonthlyCharges
* TotalCharges
* Tenure
* InternetService
* PaymentMethod

---

## Project Workflow

Dataset → Data Cleaning → Exploratory Data Analysis → KPI Generation → Dashboard Development

---

## Key Performance Indicators (KPIs)

* Total Customers
* Customer Churn Rate
* Average Monthly Charges
* Average Customer Tenure

---

## Analysis Performed

### Data Cleaning

* Checked missing values
* Checked duplicate records
* Converted TotalCharges to numeric format

### Exploratory Data Analysis

* Customer Churn Distribution
* Contract Type vs Churn
* Monthly Charges Analysis
* Customer Tenure Analysis

### Business Insights

* Customers with month-to-month contracts have the highest churn rate.
* Customers with longer tenure are less likely to churn.
* Higher monthly charges are associated with increased churn risk.
* Long-term contracts significantly improve customer retention.
